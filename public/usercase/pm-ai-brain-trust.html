痛点

PM 淹没在流程中，
而非产品

最优秀的 PM 花在实际产品思考上的时间不到 20%。其余都是文档、会议和信息搜寻。

01
PRD 永无止境的草稿
写一份扎实的 PRD 要 8–16 小时。每次利益相关方评审后更新再加 4 小时。每季度 6 个功能，成倍叠加。
02
竞品情报过时
等你整理完竞品分析，已经过时了。手动追踪 10+ 竞品的产品、定价和营销，根本不可持续。
03
会议洞察消失
用户访谈、设计评审、迭代回顾——关键洞察埋在没人读的 50 页会议纪要里。决策重复，上下文丢失。
04
用户反馈分散
反馈散落在 Intercom、Zendesk、Slack、应用商店评论和 NPS 问卷里。没有统一信号。不花一整天手动汇总，看不到规律。
05
对齐永无止境
工程要规格。设计要用户上下文。销售要产品简介。市场要定位。每个团队要同一故事的不同版本。
06
战略工作零时间
做完以上所有，没时间思考产品方向、市场趋势或 12 个月路线图。救火取代了战略。
解决方案

你的 4-Agent
PM 运营网络

每个 Agent 负责一个关键 PM 工作流。合力为你每周回收 20+ 小时——并显著提升输出质量。

🔍
MarketRadar
竞品情报 Agent
每日监控 15 个竞品——追踪产品更新、定价变化、招聘信息（战略方向信号）、新闻稿和应用商店评论。每周推送带战略含义的竞品摘要，而非原始数据。
web_search web_scrape summarize trend_detect email_send
竞品监控时间/周 8h → 0
📋
DocuBot
需求文档 Agent
接收你的粗糙功能 brief（甚至语音或 Slack 线程），生成结构化 PRD，含问题陈述、用户故事、验收标准和边缘情况。支持 Notion、Confluence 或 Linear 格式。你提供反馈后自动更新。
prd_template user_stories notion_write feedback_edit
PRD 创建时间 12h → 2h
🎙️
MeetingMind
会议智能 Agent
处理会议纪要和录音。提取行动项、关键决策、待解决问题和产品洞察。按主题标记洞察并关联到相关功能或 OKR。自动生成跟进邮件并更新 Jira 工单。
transcript_parse action_extract jira_update email_draft
会后行政时间 45min → 5min
📣
FeedbackPulse
用户反馈聚合 Agent
从 Intercom、Zendesk、应用商店评论、NPS 问卷和 Slack 客户频道拉取反馈。按主题聚类、量化频率、关联产品领域。每周推送信号报告，告诉你用户真正在说什么——而非你以为的。
intercom_pull app_store_scrape cluster_analysis theme_tag report_gen

反馈聚合时间/周 6h → 0

DocuBot — 系统提示词（节选） 
# DocuBot — PRD 生成 Agent

你从粗糙输入（Slack 线程、
语音纪要、要点）生成结构化
产品需求文档。

输出结构：
1. 问题陈述（2–3 句）
2. 用户故事（Given/When/Then）
3. 验收标准（编号列表）
4. 边缘情况与范围外
5. PM 审核待解决问题
6. 成功指标（领先 + 滞后）

格式适配 Notion API。
在缺乏足够上下文的章节
添加 [ARIA 待审核] 标签。
