Why OpenClaw

OpenClaw: An AI Army for One

Vivian had tried various tools — Coze, Dify, ChatGPT Plus — but each was either limited to single-turn Q&A or required professional development skills to build workflows. Then a friend recommended OpenClaw (community nickname "Crayfish" 🦞) — the fastest-growing open-source AI Agent project on GitHub in 2026, with over 188,000 stars.

Three qualities that won her over:

First, multi-Agent isolated execution. A single Gateway can run multiple fully independent Agents simultaneously, each with its own identity, knowledge base, and skill set. This means the Procurement Agent never conflates conversations with the Customer Service Agent, and the Social Media Agent never leaks internal pricing.

Second, native WeCom and WhatsApp support. Through community plugins, OpenClaw can connect to both platforms at once, letting Agents truly "live" inside the tools Vivian uses every day — rather than adding yet another system to log into.

Third, Heartbeat turns AI from reactive to proactive. OpenClaw's Heartbeat feature lets Agents automatically check tasks at set intervals — for instance, scanning supplier emails every 2 hours and proactively notifying Vivian when new quotes arrive. Combined with Cron Jobs, it can auto-generate a morning briefing every day.

Five Agents in Detail

Step-by-Step Setup: Building a Luxury Buyer's AI Workflow from Scratch

The following are the 5 Agent configurations Vivian uses in production. Each Agent's core is made up of three parts: Identity Definition (SOUL.md) — who it is, how it speaks, and where its boundaries lie; Skill Configuration (Skills) — what it can do; and Channel Bindings (Bindings) — where it operates.

First, define the multi-Agent routing rules in ~/.openclaw/openclaw.json:

{
  "agents": {
    "list": [
      { "id": "procurement",   "workspace": "~/.openclaw/ws-procurement" },
      { "id": "customer-svc",  "workspace": "~/.openclaw/ws-customer", "default": true },
      { "id": "social-media",  "workspace": "~/.openclaw/ws-social" },
      { "id": "lead-gen",      "workspace": "~/.openclaw/ws-leads" },
      { "id": "trade-assist",  "workspace": "~/.openclaw/ws-trade" }
    ]
  },
  "bindings": [
    { "agentId": "procurement",  "match": { "channel": "whatsapp" } },
    { "agentId": "customer-svc", "match": { "channel": "wecom" } },
    { "agentId": "social-media", "match": { "channel": "telegram" } },
    { "agentId": "procurement",  "match": { "channel": "email" } }
  ]
}

The routing logic is simple: Messages from WhatsApp and email automatically route to the "Procurement Agent," WeCom messages route to the "Customer Service Agent," and Telegram serves as a personal command console for the Social Media Agent. The first matching rule takes effect.

01

Procurement Agent

Automated Supplier Follow-up · Quote Comparison · Inventory Alerts
📜 SOUL.md Identity Definition
You are a professional luxury goods procurement assistant, deeply versed in the product lines and pricing structures of Hermès, Chanel, Louis Vuitton, Dior, and similar brands. You communicate with European suppliers via WhatsApp in fluent English and French to handle inquiries. Upon receiving new quotes, automatically compare them against historical pricing and flag any anomalies exceeding a 15% deviation. You never independently commit to purchase quantities or confirm orders — all final decisions must be approved by Vivian.

Channel & Triggers

Bound to WhatsApp (supplier communications) and Email (receiving formal quotes). Heartbeat scans the email inbox every 2 hours, extracts key fields from new quotes (brand, SKU, unit price, MOQ, lead time), and pushes them to Telegram.

Key Skill Configuration

Email Read/Write Skill — Automatically parses Excel/PDF attachments from supplier quote emails and extracts structured quote data

Custom "Quote Comparison" Skill — Comparison logic defined in SKILL.md: multi-supplier sorting for the same SKU, historical price trends, RMB landed cost after FX conversion

Cron Job Morning Report — Generates "Today's Procurement To-Dos" every morning at 8 AM: unanswered inquiries, expiring quotes, orders needing follow-up

# Cron configuration example
openclaw cron add \
  --agent procurement \
  --name "Morning Procurement Report" \
  --cron "0 8 * * *" \
  --message "Generate today's procurement morning report:
    1) Summary of yesterday's new quotes
    2) Pending inquiries
    3) Quotes expiring this week
    4) EUR/CNY exchange rate movement"
02

Customer Service Agent

WeCom 24/7 Online · Smart Q&A · Human Escalation Fallback
📜 SOUL.md Identity Definition
You are a premium customer service specialist for Vivian's team, serving domestic B2B wholesale clients. Always maintain a professional, warm, and efficient tone. You can answer: authenticity verification guidance, current in-stock styles and approximate price ranges, logistics status inquiries, and return/exchange policies. You cannot answer: exact pricing to the decimal (redirect clients to Vivian for a dedicated quote), order confirmation, or discount promises. If a conversation escalates to a complaint or a major refund request, immediately notify Vivian and attach a conversation summary.

Channel & Triggers

Bound to WeCom, set as default: true (the default Agent). Uses group chat @mention mode — the bot only replies when clients @mention it in WeCom groups, to avoid disrupting normal conversations. Each client's session is independently isolated (per-channel-peer mode).

Knowledge Base Setup

Product Catalog — Organize each brand's current in-stock items as Markdown files in ws-customer/knowledge/. Format: Brand → Category → SKU/Name/Price Range/Stock Status

FAQ Document — Compile common questions and standard answers: authenticity guarantees, shipping timelines, payment methods, return policies, etc.

Webhook ERP Integration — Custom skill that calls ERP/WMS APIs to query real-time inventory and tracking numbers

03

Social Media Agent

Content Generation Hub · Xiaohongshu Posts · In-Depth Articles
📜 SOUL.md Identity Definition
You are a content strategist specializing in luxury fashion, skilled in content creation for the following platforms: Xiaohongshu (lifestyle post style, using emojis and conversational tone, under 800 characters, with hashtags); WeChat Official Account (professional in-depth content, ~3,000-character buying guides or industry insights); Moments (brief and punchy, caption under 50 characters). Based on new product information and images from Vivian, automatically generate platform-appropriate content drafts. Be mindful of each platform's sensitive keyword policies.

Workflow

Vivian sends instructions to this Agent via Telegram. For example, she might send a photo of a newly arrived Hermès Birkin bag with the note "Generate a Xiaohongshu + Moments content set for this," and the Agent will produce two versions in different styles. A Cron Job also reminds Vivian every Wednesday and Saturday at 2 PM (before peak Xiaohongshu traffic hours) to review the pending content queue.

# Scheduled reminder configuration
openclaw cron add \
  --agent social-media \
  --name "Content Publishing Reminder" \
  --cron "0 14 * * 3,6" \
  --message "Review this week's pending content queue and remind Vivian to approve and publish"

Advanced Usage

Webhook → n8n → Auto-publish — After OpenClaw generates the copy, trigger an n8n workflow via Webhook to automatically call each platform's API for publishing (requires your own n8n setup)

Competitor Content Analysis — Use the browser skill to periodically scrape top-performing Xiaohongshu posts from competitors, analyzing their titles, hashtags, and engagement metrics for content strategy inspiration

Lead Intelligence Agent

Market Intelligence · Client Identification · Trend Prediction
📜 SOUL.md Identity Definition
You are a market intelligence analyst focused on buyer communities and emerging retail channels in China's luxury wholesale market. You periodically scan industry sources to identify potential B2B client leads (newly opened boutiques, social signals indicating procurement intent), and analyze market trends and competitive dynamics. Output format is standardized: lead source, contact/channel, inferred potential needs, and recommended follow-up approach.

Automation Logic

This is a "silent background Agent" — not bound to any messaging channel. It operates automatically via Heartbeat and Cron Jobs. Every 4 hours it scans industry websites and social platforms using the browser skill, and every Friday afternoon generates a "Weekly Market Intelligence Briefing" delivered via Telegram.

05

Cross-Border Trade Assist Agent

FX Monitoring · Tariff Lookup · Logistics Tracking

This is not a standalone Agent — it functions as a shared skill layer serving other Agents. Vivian places the following three custom skills in the shared directory ~/.openclaw/skills/, making them available to all Agents:

FX Monitoring Skill — Retrieves real-time EUR/CNY and USD/CNY exchange rates via API. Proactively pushes an alert when daily fluctuation exceeds 0.5%. The Procurement Agent calls this automatically when calculating landed costs

HS Code Lookup Skill — Input a product description (e.g., "French-made women's leather handbag"), automatically match the HS tariff code and corresponding import duty rate. Reduces customs declaration errors

Logistics Tracking Skill — Integrates the 17Track API to query international parcel statuses in a unified interface. The Customer Service Agent calls this automatically when answering client shipping inquiries

# Daily FX report Cron configuration
openclaw cron add \
  --agent trade-assist \
  --name "Daily FX Report" \
  --cron "0 9 * * 1-5" \
  --message "Query today's EUR/CNY and USD/CNY exchange rates,
    compare with yesterday. If fluctuation exceeds 0.5%, flag with a red alert."

Implementation Path

From Installation to Full Operation: A 4-Week Step-by-Step Plan

Vivian's key lesson: never go live with all Agents at once. She recommends the following pace — launch one module per week, ensuring the previous one is stable before adding the next.

Week 1 · Foundation
Install OpenClaw + Launch Customer Service Agent

Run curl -fsSL https://openclaw.ai/install.sh | bash to install Gateway, complete initialization via openclaw onboard. Install the WeCom plugin openclaw plugins install @openclaw-china/wecom, configure the customer service Agent's SOUL.md and product knowledge base. This is the highest-ROI entry point — it immediately relieves pressure from client inquiries.

Week 2 · Core Business
Launch Procurement Agent

Connect the WhatsApp channel and email skill, configure quote parsing and comparison skills. Let the Agent run in "read-only" mode for the first week — only organizing information and sending notifications, without auto-replying to suppliers. After verifying accuracy, gradually expand auto-reply permissions.

Week 3 · Growth Engine
Launch Social Media Agent + Cross-Border Assist Skills

Configure Telegram as the content command center, test Xiaohongshu and WeChat article copy generation quality. Simultaneously deploy the FX/tariff/logistics shared skill layer so both the Procurement and Customer Service Agents can access them.

Week 4 · Smart Expansion
Launch Lead Intelligence Agent + Full Optimization

Deploy the background intelligence Agent, configure Heartbeat and Cron Jobs. Review the first three weeks' operation logs, fine-tune each Agent's SOUL.md boundary definitions and skill parameters. Set up the ClawRouter skill to auto-switch models based on task complexity and optimize API costs.

Security & Cost

Vivian's Hard-Learned Lessons — So You Don't Have To

⚠️ Security First: These Lines Must Not Be Crossed

Never expose Gateway to the public internet. Bind to 127.0.0.1:18789 by default; use Tailscale or SSH tunnels for remote management. A WebSocket hijacking vulnerability (CVE-2026-25253) was disclosed in January 2026 — instances exposed to the internet are at extreme risk.

Audit community skills before installing. Security researchers found that approximately 36% of community skills contain potentially risky code. Always manually review each new skill's SKILL.md file before enabling it.

Agents must not have write access to core systems. Letting an Agent query ERP inventory is fine — but don't grant it permissions to create orders or modify pricing. Enable exec_approval for all high-risk operations requiring human confirmation.

Create minimum-privilege accounts for each Agent. Assume that anything an Agent can see could potentially be exfiltrated via prompt injection. Never place sensitive information like personal messaging accounts or banking credentials in an Agent's workspace.
