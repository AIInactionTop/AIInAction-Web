Two Core Agents: In Depth

Building Step by Step: A Round-the-Clock AI Security Workflow

Simon leveraged OpenClaw’s modular design and Agent system to carefully build a personal round-the-clock AI assistant framework. He deployed two core Agents: a customized AgentX (AI Security Assistant) and an Email Management Agent.

01

AgentX Security Assistant

24/7 Network Monitoring · Penetration Testing Support · Auto-Generated Reports & Scripts
📜 Core Function
AgentX is Simon’s round-the-clock AI security deputy, responsible for network scanning, WiFi monitoring, penetration testing support, and the automated generation of security reports and scripts. In Heartbeat mode, it periodically performs network topology discovery and WiFi security audits, instantly pushing alerts through Telegram the moment anomalies are detected.

Skill Integrations

Shell Skill — Execute network scanning tools like Nmap and Masscan, as well as custom security scripts, enabling automated vulnerability detection and configuration auditing

File Read/Write Skill — Read scan results and log files, output analysis reports in PDF format, manage and update the penetration testing script library

web_search Skill — Gather the latest threat intelligence and research CVE vulnerability details for non-sensitive projects (disabled for sensitive engagements)

Core Tasks

Network Scanning & WiFi Monitoring — Periodically executes network topology discovery and WiFi security audits in Heartbeat mode, pushing Telegram alerts when unauthorized devices connect or port changes are detected

Penetration Testing Support — Invokes the Shell Skill to automatically execute attack scripts on command, records attack paths and results, dramatically reducing penetration test preparation and execution time

Report & Script Development — Automatically generates structured security report PDFs from collected data, assists with writing and debugging Python security scripts, providing code suggestions and bug fixes

Channels & Triggers

Telegram serves as AgentX’s primary output channel. Simon receives real-time security event alerts and task progress updates, and can control AgentX’s actions through simple conversational commands.

02

Email Management Agent

Smart Classification · Auto-Draft Replies · Real-Time Priority Alerts
📜 Core Function
The Email Management Agent is laser-focused on solving Simon’s most painful email management problem. It accesses the inbox with read-only permissions to ensure data security. Through smart classification, auto-draft replies, and priority email alerts, it compresses daily email handling from 30–45 minutes down to just 5–10 minutes.

Skill Integrations

Email API (Gmail/Outlook, read-only) — Accesses Simon’s inbox with read-only permissions, ensuring data security with no risk of tampering

Core Tasks

Smart Classification — Automatically sorts emails based on content, sender, and subject into custom folders like “Client Urgent”, “Project Notifications”, “Internal Updates”, and “Potential Spam”

Auto-Draft Replies — For common inquiries, automatically generates draft responses based on past emails and Simon’s preferences, ready for review before sending

Priority Email Alerts — Checks the inbox every 30 minutes in Heartbeat mode, sending immediate Telegram notifications when urgent emails or emails from specific senders arrive

Configuration Example

YAML Configuration Code Reference

# OpenClaw AgentX Configuration Snippet
agent:
  name: "AgentX"
  description: "Round-the-clock AI security assistant for network monitoring, scanning, and report generation."
  model: "claude-3-opus-20240229" # or gpt-4-turbo
  skills:
    - name: "shell"
      description: "Execute system shell commands, e.g. nmap, masscan, python scripts."
      permissions:
        - "nmap *"
        - "masscan *"
        - "python3 /path/to/scripts/*"
    - name: "file_rw"
      description: "Read/write files for report generation and script management."
      permissions:
        - "read /var/log/security/*"
        - "write /home/simon/reports/*.pdf"
        - "read /home/simon/scripts/*"
        - "write /home/simon/scripts/*"
    - name: "web_search"
      description: "Fetch information via web search (enabled for non-sensitive projects)."
  channels:
    - type: "telegram"
      chat_id: "your_telegram_chat_id"
  heartbeat:
    interval_minutes: 60 # Automatically check network and tasks every hour
# Email Management Agent Configuration Snippet
agent:
  name: "Email Management Agent"
  description: "Automated email classification, draft generation, and priority alerts."
  model: "claude-3-haiku-20240307" # cost-effective model
  skills:
    - name: "email_api"
      description: "Access Gmail/Outlook inbox (read-only permission)."
      permissions:
        - "read_inbox"
        - "classify_email"
        - "create_draft"
  channels:
    - type: "telegram"
      chat_id: "your_telegram_chat_id"
  heartbeat:
    interval_minutes: 30 # Check for new emails every 30 minutes

Reusable Solutions

Five Practical Tips to Replicate Simon’s Success

1

Start Small, Focus on Pain Points

First, identify the 1–2 most time-consuming and repetitive tasks in your daily workflow (e.g., email management, meeting notes), then build simple OpenClaw Agents for them. Expand gradually rather than trying to automate everything at once.

2

Embrace the “Heartbeat” Proactive Mode

This is one of OpenClaw’s killer features. Use Heartbeat mode to let your Agents periodically “wake up” and execute tasks (e.g., checking new emails, monitoring GitHub, fetching news), transforming reactive responses into proactive management.

3

Leverage Skills & the Community Ecosystem

Explore community-contributed skill libraries, or customize your own skills (e.g., Shell, API calls) to extend your Agent’s capabilities. OpenClaw’s Skills ecosystem is one of its greatest strengths.

4

Prioritize Security & Privacy

For scenarios involving sensitive data or operations, always leverage OpenClaw’s multi-Agent isolation, security sandboxing, and granular permission controls. Consider local/self-hosted deployment to maximize data privacy and control.

5

Multi-Model & Deterministic Workflows

OpenClaw supports selecting different LLMs (e.g., GPT, Claude, Gemini) for each step in your workflow, allowing optimization based on task complexity and cost. Use deterministic, node-based workflow design to build automation that is far more predictable and reliable than traditional chatbots.
