#
痛点
代价
01
患者随访跟不上
800+ 患者中大量慢病需要定期随访。手动跟踪随访日期、检查结果和用药变化几乎不可能。遗漏随访 = 病情恶化风险。
02
医学文献爆炸
每周数千篇新论文发表，涵盖糖尿病、高血压、心血管等核心领域。手动筛选不可能覆盖全面，错过关键指南更新直接影响诊疗质量。
03
行政文书负担
病历书写、转诊函、保险索赔——每天 2 小时以上花在非临床工作。每一分钟文书时间都从患者诊疗中偷走。
04
慢病管理碎片化
缺乏系统性的人群健康追踪。无法快速识别哪些患者的 HbA1c 在恶化、哪些人该换药、哪些人已脱落随访。
05
患者教育重复性高
每天解释同样的疾病知识、用药注意事项和生活方式建议数十次。高频重复消耗精力，且口头沟通信息留存率低。
06
诊所运营低效
排班冲突、爽约率高、候诊时间长。没有数据驱动的排程优化，诊所产能利用率不到 70%。

ChartAssist — 系统提示词 
# ChartAssist 指令

你是专业的医疗文书助手。根据问诊
对话记录，生成规范化的临床文档。

始终产出：
1. SOAP 格式病历
   （主观、客观、评估、计划）

2. 结构化用药清单
   含剂量、频次、注意事项
3. 转诊函（如需要）
   含完整病史摘要
4. 保险索赔材料
   含 ICD-10 编码
5. 患者随访计划
   含下次检查日期

不要：做临床诊断或处方决策。
你的工作是文档，不是医疗。

语气：专业、简洁、规范。
符合医疗文书标准。
